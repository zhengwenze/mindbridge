# MindBridge Frontend Design System

本文档定义 MindBridge 前端第一版设计系统。后续页面必须遵循本文档，避免学生端和管理端视觉割裂。

## 1. 设计原则

MindBridge 是校园心理智能体平台，不是普通 CRUD 后台。

设计目标：

- 可信：界面稳定、清晰、不过度装饰。
- 温和：学生端避免压迫感和医疗化视觉。
- 高效：管理端支持快速扫描风险、报告、个案和工具状态。
- 克制：不使用夸张渐变、装饰大卡片、花哨动效。
- 一致：所有状态、风险、时间、按钮、表格行为保持统一。

## 2. 产品区域

### 2.1 学生端

关键词：

- 温和
- 安静
- 陪伴
- 安全
- 低干扰

学生端重点是“愿意表达”，不是展示系统能力。

避免：

- 大面积红色。
- 明确展示风险评分。
- 展示后台 Agent 判断链路。
- 使用冰冷的医疗化语言。
- 过多运营指标。

### 2.2 管理端

关键词：

- 高密度
- 可扫描
- 可追踪
- 可审计
- 可闭环

管理端重点是“快速发现高风险并完成处置闭环”。

需要突出：

- 风险等级。
- 个案状态。
- 最近更新时间。
- 工具任务失败。
- 会话档案。
- Agent trace。

## 3. 色彩系统

第一版采用亮色主题。

### 3.1 品牌色

- Primary: 稳定蓝绿色，用于主按钮、链接、选中状态。
- Info: 中性蓝，用于系统信息。
- Success: 绿色，用于成功、低风险、正常状态。
- Warning: 橙色，用于中风险、处理中、等待。
- Error: 红色，用于高风险、失败、严重错误。

### 3.2 风险色

- LOW: 绿色，表示低风险。
- MEDIUM: 橙色，表示中风险。
- HIGH: 红色，表示高风险。
- CRITICAL: 深红，表示极高风险或紧急状态。

规则：

- 学生端不直接展示风险色标签。
- 管理端可以清晰展示风险色。
- 红色只用于真实风险和错误，不用于普通强调。
- 风险色必须配合文字，不只依赖颜色。

### 3.3 背景色

- 页面背景：浅灰白。
- 内容区：白色。
- 边框：浅灰。
- 分割线：低对比灰。
- 悬浮背景：极浅蓝灰。

禁止：

- 大面积深色背景。
- 大面积紫蓝渐变。
- 大面积暖黄、米色、棕色风格。
- 装饰性光斑、渐变球、复杂背景纹理。

## 4. 字体与排版

### 4.1 字体

使用系统字体栈，优先保证中文可读性和跨平台稳定。

### 4.2 字号层级

- 页面标题：20-24
- 区块标题：16-18
- 表格正文：13-14
- 辅助信息：12-13
- 标签文字：12

规则：

- 管理后台不要使用超大标题。
- 页面标题保持克制。
- 表格、详情、抽屉内文字优先可读。
- 不使用负 letter-spacing。
- 不使用随视口缩放的字体。

## 5. 间距系统

使用 4px 基础间距体系：

- 4: 极小间距
- 8: 控件内部或紧凑间距
- 12: 表单项间距
- 16: 常规模块间距
- 24: 页面区块间距
- 32: 大区块间距

页面布局规则：

- 管理端优先 16 和 24。
- 学生端可以适度增加留白。
- 卡片内部不要堆过多层级。
- 不使用卡片套卡片。

## 6. 布局规范

### 6.1 应用骨架

管理端：

- 左侧导航。
- 顶部状态栏。
- 主内容区。
- 详情使用 Drawer。

学生端：

- 顶部品牌与状态。
- 左侧快捷表达和说明。
- 主聊天区。
- 移动端左侧内容折叠到上方或抽屉。

### 6.2 页面容器

统一页面结构：

```text
Page
-> PageHeader
-> Toolbar or SearchForm
-> Main Content
-> Drawer or Modal
```

页面不要做营销式 hero。MindBridge 第一屏应该直接进入产品工作台。

## 7. 组件规范

### 7.1 Button

使用 Ant Design Button。

规则：

- 主操作使用 primary。
- 次操作使用 default。
- 危险操作使用 danger。
- 图标按钮需要 Tooltip。
- 同一区域主按钮最多一个。

### 7.2 Table

使用 Ant Design Table。

要求：

- 必须有 loading。
- 必须有 empty state。
- 必须有错误反馈。
- 操作列固定在右侧时保持宽度稳定。
- 长文本使用省略和详情 Drawer。

### 7.3 Tag

所有状态使用统一 Tag。

风险：

- LOW
- MEDIUM
- HIGH
- CRITICAL

任务：

- PENDING
- RUNNING
- SUCCESS
- FAILED
- DEAD_LETTER

个案：

- OPEN
- ACKNOWLEDGED
- FOLLOWING
- CLOSED

### 7.4 Drawer

详情优先使用 Drawer：

- 报告详情。
- 会话档案。
- Agent trace。
- 工具错误详情。
- 个案详情。

Drawer 适合后台不中断列表上下文。

### 7.5 Modal

Modal 只用于确认类和短表单：

- 删除确认。
- 重建向量索引确认。
- 退出确认。
- 简短备注。

### 7.6 Upload

知识库上传使用 Ant Design Upload。

要求：

- 显示支持格式。
- 显示上传中状态。
- 显示成功入库片段数。
- 显示失败原因。

## 8. 学生端设计规范

学生端核心是聊天。

### 8.1 聊天气泡

- 学生消息右侧。
- MindBridge 消息左侧。
- 学生消息可使用浅主色。
- Assistant 消息使用白色或浅灰。
- 保持足够行高。

### 8.2 输入区

- textarea 多行输入。
- 发送中禁用按钮。
- 支持新会话。
- 不显示复杂参数。

### 8.3 快捷表达

快捷表达用于降低开口难度：

- 压力失眠。
- 焦虑倾诉。
- 低落求助。
- 关系困扰。

文案要温和，不要像诊断问卷。

### 8.4 风险信息隐藏

学生端禁止展示：

- riskLevel。
- emotionScore。
- confidence。
- 后台报告。
- Agent trace。
- Tool audit。

## 9. 管理端设计规范

管理端核心是风险发现和处置闭环。

### 9.1 首页指标

展示：

- 报告数。
- 高风险数。
- 个案数。
- Excel 台账。
- 预警记录。

指标卡保持紧凑，不做大面积装饰。

### 9.2 报告列表

重点字段：

- 学生。
- 风险等级。
- 意图。
- 情绪。
- 摘要。
- 创建时间。
- 操作。

点击报告后打开会话档案 Drawer。

### 9.3 个案列表

重点字段：

- 个案 ID。
- 风险等级。
- 状态。
- 负责人。
- 摘要。
- 更新时间。

高风险个案需要醒目但不刺眼。

### 9.4 Agent Trace

Agent Trace 是技术展示重点。

推荐展示：

- timeline。
- agent steps。
- retrieved knowledge。
- assessment。
- response messages。

避免直接塞 JSON 大段文本。JSON 只作为展开详情。

## 10. 响应式规范

支持桌面和移动端。

优先级：

- 管理端优先桌面体验。
- 学生端必须移动端可用。

移动端规则：

- 管理端表格可横向滚动。
- 详情 Drawer 改为全屏或接近全屏。
- 学生端输入区固定在底部时不能遮挡消息。
- 页面文本不得溢出按钮或卡片。

## 11. 动效规范

动效只用于状态反馈：

- loading。
- drawer 打开。
- message streaming。
- button hover。

禁止：

- 大面积入场动画。
- 装饰性漂浮动画。
- 无业务意义的复杂动效。

## 12. 可访问性

要求：

- 按钮有明确文本或 Tooltip。
- 图标按钮必须可理解。
- 表单错误清晰展示。
- 风险颜色必须配合文字。
- 键盘可操作主要表单。

## 13. Ant Design 与 Tailwind 边界

Ant Design 控件不做深度样式重写。

Tailwind 只做：

- 布局。
- 间距。
- 宽高。
- 响应式。

主题色、按钮色、表格色、状态色由 Ant Design token 和统一字典控制。
