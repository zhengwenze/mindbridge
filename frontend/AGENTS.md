# MindBridge Agent Instructions

## Frontend design

Before creating or changing any frontend page, component, layout, or visual style, read [`DESIGN.md`](./DESIGN.md) completely and treat it as the project's design source of truth.

- Reuse its tokens, spacing scale, page structures, component patterns, responsive rules, accessibility requirements, and student-safety boundaries.
- Do not introduce a second visual language or deviate from `DESIGN.md` without explicit user approval.
- Keep admin pages task-first and compact; do not restore redundant breadcrumbs, large page titles, or page descriptions.
- Keep student experiences calm and low-stimulation; never expose backend risk assessments or Agent reasoning.
