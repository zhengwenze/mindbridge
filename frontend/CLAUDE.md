# MindBridge 项目协作约束

## Design System

进行任何前端页面、组件、布局或视觉修改前，必须完整阅读 `frontend/` 目录中的 [`DESIGN.md`](./DESIGN.md)。

- 字体、颜色、间距、布局、圆角、状态、响应式和心理安全规则均以 `DESIGN.md` 为准。
- 未经用户明确确认，不得偏离或创建第二套设计系统。
- 新管理页面默认直接进入业务内容，不重复渲染面包屑、大页面标题和说明段。
- 学生端不得展示后台风险评估、情绪评分、报告或 Agent 决策链路。
- UI 评审时应把不符合 `DESIGN.md` 的实现视为需要修正的问题。
